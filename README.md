## 🏆 Engineering proof of work

<table align="center" width="100%" border="0" cellpadding="15" cellspacing="15">
  <tr>
    <td width="50%" align="center" valign="top">
      <h3>🛡️ <a href="https://github.com/Mentorzx/formae">Formaê</a></h3>
      <a href="https://mentorzx.github.io/formae/">
        <img src="./assets/profile/formae-dashboard.png" width="100%" alt="Formae live interface">
      </a>
      <p align="left">
        <b>Problem:</b> academic planning and SIGAA import flows involve sensitive student data.<br>
        <b>Architecture:</b> local-first UFBA assistant with <b>React/Vite PWA</b>, <b>MV3 extension</b> sync and shared logic in <b>Rust compiled to WASM</b>.<br>
        <b>Why it matters:</b> privacy stays in the browser, while the product still delivers search, planning, parsing and import automation.
      </p>
    </td>
    <td width="50%" align="center" valign="top">
      <h3>🧠 <a href="https://github.com/Mentorzx/MCP-register">MCP-register</a></h3>
      <a href="https://github.com/Mentorzx/MCP-register">
        <img src="./assets/profile/cards/mcp-register-card.svg" width="100%" alt="MCP-register architecture card">
      </a>
      <p align="left">
        <b>Problem:</b> exact-match storage is weak when the query is driven by meaning instead of plain text.<br>
        <b>Architecture:</b> Python MCP server combining <b>SQLite</b> as the source of truth, <b>FAISS</b> as the derived vector index and local embeddings via <b>sentence-transformers</b>.<br>
        <b>Why it matters:</b> shows semantic retrieval, modular backend slices and clear runtime boundaries without leaning on external APIs.
      </p>
    </td>
  </tr>
  <tr>
    <td width="50%" align="center" valign="top">
      <h3>🎵 <a href="https://github.com/Mentorzx/AudCifra">AudCifra</a></h3>
      <a href="https://github.com/Mentorzx/AudCifra">
        <img src="./assets/profile/cards/audcifra-card.svg" width="100%" alt="AudCifra pipeline card">
      </a>
      <p align="left">
        <b>Problem:</b> raw music audio is messy input when the goal is a usable chord-and-lyrics document.<br>
        <b>Architecture:</b> Python workflow for transcription, chord detection, alignment and <b>DOCX</b> export, with concurrent execution through <b>asyncio</b> and <b>ThreadPoolExecutor</b>.<br>
        <b>Why it matters:</b> connects audio processing, structured output generation and pipeline orchestration in one artifact.
      </p>
    </td>
    <td width="50%" align="center" valign="top">
      <h3>📨 <a href="https://github.com/Mentorzx/Hermes">Hermes</a></h3>
      <a href="https://github.com/Mentorzx/Hermes">
        <img src="./assets/profile/cards/hermes-card.svg" width="100%" alt="Hermes API card">
      </a>
      <p align="left">
        <b>Problem:</b> academic NLP experiments often stop at notebooks and never become something queryable by other systems.<br>
        <b>Architecture:</b> <b>Flask</b> API around SVM-based sentiment and figurative-language classification over tweet datasets, with keyword search and trend endpoints.<br>
        <b>Why it matters:</b> shows classical ML exposed as software, not just a one-off academic script.
      </p>
    </td>
  </tr>
  <tr>

        <img src="./assets/profile/cards/production-fix-flow-card.svg" width="100%" alt="Production Fix Flow architecture card">
      </a>

        <b>Problem:</b> operational API sequences get hard to audit, rerun and observe when orchestration logic spreads across ad-hoc scripts.<br>
        <b>Architecture:</b> large orchestration codebase mixing <b>CLI</b>, <b>FastAPI</b> and worker flows, with <b>Redis</b>, <b>PostgreSQL</b>, typed config, rate limiting, structured logging and containerized runtime pieces.<br>
        <b>Why it matters:</b> demonstrates runtime boundaries, validation discipline and production-minded backend decomposition in a non-trivial codebase.
